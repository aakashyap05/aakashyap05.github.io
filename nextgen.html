Overview

At NextGen Federal Systems, I support simulation-focused digital engineering work at the intersection of aerospace engineering, software development, simulation tools, and repeatable technical workflows. My experience includes Ansys modeling, Ansys Fluent simulation, STK scenario support, Python scripting, C++ development, Docker-based environments, and WSL/Linux workflows.

This work has strengthened my ability to think beyond a single analysis tool. I have been able to connect geometry/modeling, simulation setup, performance estimation, scenario inputs, automation, and documentation into workflows that support larger engineering objectives.

Aerospace Modeling and Simulation

Before moving into my current containerized simulation workflow support, I worked with Ansys products to build my understanding of aerospace simulation tools, including Ansys Fluent, Ansys Motion, and finite element analysis software.

One major simulation effort involved modeling a KC-135 aircraft in Ansys and running a Fluent-based analysis to support fuel consumption estimation. The resulting engineering output was then used as an input for an STK scenario, connecting vehicle-level simulation work to mission/scenario-level analysis.

This experience helped me understand how CFD and simulation results can feed into broader aerospace workflows, where engineering estimates are not isolated numbers but inputs that affect larger system behavior and mission planning.

Workflow Automation Contribution

One of my key contributions was developing a Python workflow script to streamline VDB generation after a container has been built. The script was created to make the workflow easier to run, reduce manual setup steps, and lower the chance of user error when converting TIFF data into VDB files.

This work gave me practical experience with automation, container-based workflows, file conversion processes, and improving the usability of technical tools for engineering and simulation tasks.
